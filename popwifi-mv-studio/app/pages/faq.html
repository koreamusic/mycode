사용 설명 / 개발 가이드

프리셋 제작법, 삭제/비활성화, 롱폼 재설계 규칙, 문제 해결을 안내합니다.

전체 제작 흐름

  1. 프로젝트 페이지에서 싱글MV / 롱폼 / 숏츠 선택
  2. 롱폼 또는 숏츠 페이지에서 프리셋 선택
  3. 가사 페이지에서 타임코드 마킹 후 sync.json 내보내기
  4. 랜딩 검수 페이지에서 720p 30초 테스트 렌더
  5. 확인 후 렌더 큐에 등록 → FFmpeg 출력

프리셋 파일 구조

각 프리셋 폴더는 반드시 4개 파일을 가집니다.

shared/presets/
  9x16/batch-001-010/
    my-preset/
      config.json   ← 프리셋 메타 및 설정
      template.js   ← Canvas 렌더 함수 (CommonJS)
      preview.jpg   ← 썸네일 (선택)
      README.md     ← 재설계 원칙 메모

프리셋 추가 방법

  1. 위 폴더 구조대로 파일 생성
  2. 설정 페이지 → 프리셋 배치 가져오기에서 JSON 파일 업로드
  3. 또는 shared/presets/imports/에 배치 JSON 파일 추가 후 서버 재시작

배치 JSON 형식: {"ratio":"16x9","batchId":"batch-001-010","presets":[...]}

프리셋 비활성화 vs 삭제

비활성화: 프리셋 카드 우측 하단 비활성 버튼 클릭.
목록에서 숨겨지며 config.json에 "active": false가 기록됩니다.
언제든 복원 가능합니다.

삭제: 서버에서 폴더를 직접 삭제합니다.
shared/presets/9x16/batch-001-010/my-preset/ 폴더 전체 삭제 후 서버 재시작.

숏츠 → 롱폼 재설계 규칙

  • 숏츠 9x16은 디자인 원형이다
  • 롱폼 16x9는 숏츠를 단순 확대하면 실패한다
  • 타이틀 위치: 수직 중앙 → 수평 좌측 앵커로 재배치
  • 프레임: 세로 단일 박스 → 가로 듀얼 구도로 재설계
  • 배경/텍스처: 세로 흐름 → 수평 흐름으로 전환
  • config.json에 sourceRole, sourcePreset 필드 필수

문제 해결

렌더 실패: 렌더 큐 페이지에서 실패 작업 확인 → output/*/render.log 검토 → FFmpeg 경로 설정 페이지에서 재확인.

프리셋 목록 안 보임: 새로고침 버튼 클릭 또는 active: false인지 확인.

가사 싱크 틀림: 가사 페이지에서 타임코드 재마킹 후 sync.json 다시 내보내기.

포트 충돌: config.json의 port 값 변경 후 서버 재시작.

개발 원칙 요약

  • PC 중심 제작, 모바일은 확인용
  • 원페이지 구조 금지 — 모든 기능은 독립 모듈
  • 숏츠 프리셋 단순 확대 적용 금지
  • 사이드바/주요 기능 임의 삭제 금지
  • 프리셋 추가를 위해 앱 전체 구조 재설계 금지
