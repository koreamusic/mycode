# Pop WiFi MV Studio — PC First Architecture Plan

작성일: 2026-04-25
상태: 기준 확정

---

## 1. 개발 기준

Pop WiFi MV Studio는 PC 중심 제작 스튜디오다.

모바일은 확인용이다.

절대 모바일 중심으로 개발하지 않는다.

---

## 2. 앱 구조

하나의 앱 안에 여러 페이지/모듈을 둔다.

원페이지 몰빵 구조는 금지한다.

```text
app/
  shell/
    topbar
    sidebar
    layout

  pages/
    project
    intro-longform
    intro-shorts
    lyric-marker
    landing-review
    render-queue
    settings
    faq

  shared/
    state
    components
    presets
    renderer-bridge
    file-utils
```

## 3. 페이지별 역할

### 프로젝트

- 새 프로젝트 생성
- 싱글 / 롱폼 / 숏츠 타입 선택
- 음원, 이미지, 가사 파일 연결
- 프로젝트 저장
- 최근 프로젝트 표시

### 롱폼 인트로 프리셋

- 16:9 프리셋 관리
- 숏츠 기반으로 재설계된 PC용 프리셋 사용
- 제목/CTA/하단바 미리보기
- 프리셋 추가/삭제/비활성화

### 숏츠 프리셋

- 9:16 프리셋 관리
- 업로드된 숏츠 프리셋 기준 사용
- 모바일 확인용 프리뷰 중요
- 프리셋 추가/삭제/비활성화

### 가사 마킹

- Whisper 자동 마킹
- 수동 보정
- 재생/정지/되감기/앞으로 이동
- 라인별 시작/끝 시간 수정

### 랜딩 검수

- 720p 30초 테스트
- 720p 1분 테스트
- 최종 렌더 전 확인
- 프리셋/가사/하단바/인트로 동작 검수

### 렌더 큐

- 작업 대기열 확인
- 렌더 진행률 확인
- 취소/재정렬/삭제
- 완료 영상 확인

### 설정

- FFmpeg 경로
- 폰트 관리
- 출력 폴더
- 자동 절전
- 렌더 품질

### FAQ

- 전체 구조 설명
- 프리셋 제작법
- 숏츠→롱폼 재설계 규칙
- 문제 해결
- Codex 인수인계 안내

## 4. PC 레이아웃 기준

PC 화면은 작업 효율을 우선한다.

추천 구조:

```text
┌──────────────────────────────┐
│ Topbar                       │
├──────────┬───────────────────┤
│ Sidebar  │ Main Work Area    │
│          │                   │
│          │ Preview + Panels  │
└──────────┴───────────────────┘
```

PC에서는 다음을 지켜야 한다.

- 미리보기는 크고 명확하게 표시
- 설정 패널은 오른쪽 또는 하단에 배치
- 사이드바는 주요 페이지 이동만 담당
- 긴 설명은 본문에 노출하지 말고 FAQ/팝업으로 이동
- 프리셋 선택은 카드/필터 구조로 제공
- 한 화면에 너무 많은 안내 문구를 넣지 않는다

## 5. 모바일 기준

모바일은 확인용이다.

모바일에서 가능한 기능:

- 프로젝트 상태 확인
- 숏츠 화면 확인
- 렌더 결과 확인
- 간단한 프리셋 미리보기

모바일에서 우선하지 않는 기능:

- 복잡한 프로젝트 생성
- 대량 프리셋 관리
- 정교한 가사 마킹
- 렌더 큐 세부 제어

## 6. 공통 상태 관리

공통 상태 예시:

```json
{
  "activeProjectId": null,
  "activeRatio": "16x9",
  "activeLongformPreset": null,
  "activeShortsPreset": null,
  "activeTrackId": null,
  "lyricsStatus": "empty",
  "renderStatus": "idle"
}
```

각 페이지는 이 공통 상태를 읽되, 자기 페이지 내부 UI 상태는 별도로 가진다.

---

## 7. 업데이트 안전 원칙

- 한 기능 수정이 다른 페이지를 깨면 실패다.
- 사이드바/탑바/공통 레이아웃은 보호 구역이다.
- 프리셋 추가는 앱 구조 변경 없이 가능해야 한다.
- 기존 안정 프리셋은 직접 덮어쓰지 않는다.
- 문서 업데이트 없이 구조 변경 금지.
