# Pop WiFi MV Studio — Shorts to Longform Redesign Rules

작성일: 2026-04-25
상태: 기준 확정

---

## 1. 핵심 원칙

숏츠 프리셋은 Pop WiFi MV Studio의 핵심 디자인 원형이다.

롱폼 16:9 프리셋은 숏츠 프리셋을 단순 변환하지 않는다.
반드시 16:9 화면에 맞게 재설계한다.

```text
숏츠 프리셋 = 디자인 원형
롱폼 프리셋 = 같은 감성을 가진 16:9 재설계본
```

---

## 2. 그대로 가져올 것

숏츠 프리셋에서 유지해야 할 요소:

- 프리셋 이름
- 장르/무드
- 색감
- 액자 프레임의 핵심 형태
- 제목 연출의 성격
- CTA 디자인 언어
- 하단바 분위기
- 파티클/애니메이션 방향
- 시각적 아이덴티티

---

## 3. 새로 설계할 것

롱폼에서 새로 설계해야 할 요소:

- 제목 위치
- 제목 크기
- CTA 위치
- CTA 크기
- 하단바 높이
- 하단바 정보 배치
- 좌우 여백 사용 방식
- 파티클 밀도
- 프레임 두께
- 화면 중심축
- 미리보기 썸네일 카드 비율

---

## 4. 절대 금지

다음은 실패 처리한다.

- 9:16 좌표를 16:9에 그대로 적용
- 숏츠 프리셋을 가로로 늘리기
- 프레임을 단순 확대/축소
- 제목과 CTA 겹침
- 하단바가 지나치게 두꺼움
- 화면이 프레젠테이션처럼 보임
- 세로형 중심 구도를 가로형에 억지로 배치
- 기존 사이드바/페이지 구조를 프리셋 때문에 변경

---

## 5. 타임라인 유지 규칙

숏츠와 롱폼 모두 아래 흐름을 유지한다.

```text
0~5초:
- 액자 프레임
- 제목

5~10초:
- 좋아요 / 구독 / 알림 CTA

10초 이후:
- 액자와 CTA 제거
- 하단바 유지
- 가사 또는 제목 표시
- 하단바 끝 애니메이션 유지
```

---

## 6. 16:9 롱폼 배치 기준

### 제목

권장:

- 화면 중앙 또는 중앙보다 약간 위
- 좌우 여백을 살린 넓은 호흡
- 긴 제목 자동 축소
- 2줄 제목 가능

금지:

- 세로형처럼 너무 크게 중앙을 장악
- 하단바와 겹침
- CTA와 같은 위치 사용

### CTA

권장:

- 5~10초 구간에 별도 포커스
- 제목보다 약간 아래 또는 중앙 하단
- 카드 3개 분리 또는 하나의 통합 글래스 카드
- 화면 폭의 35~55% 안에서 구성

금지:

- 제목 위에 겹쳐놓기
- 화면 전체를 가리는 CTA
- 동일한 CTA 디자인을 모든 프리셋에 반복

### 하단바

권장:

- 높이: 화면 높이의 약 10~16%
- 약한 색이 들어간 투명 오버레이
- 가사 또는 제목 노출
- 끝부분에 작은 애니메이션 요소
- 좌측/우측 정보 분리 가능

금지:

- 완전 투명해서 안 보임
- 너무 두꺼워 영상 영역을 가림
- 가사 가독성이 낮음

---

## 7. 9:16 숏츠 배치 기준

숏츠는 세로 화면이므로 중심축이 중요하다.

권장:

- 제목은 세로 중앙 근처
- CTA는 제목 아래쪽
- 하단바는 너무 높지 않게
- 터치 화면에서도 읽히는 큰 가독성
- 모바일 검수에서 깨짐 없어야 함

---

## 8. 같은 이름 / 다른 구현

같은 디자인 계열은 같은 이름을 사용할 수 있다.

예:

```text
moon-dust-window
```

파일 구조:

```text
shared/presets/9x16/moon-dust-window/template.js
shared/presets/16x9/moon-dust-window/template.js
```

주의:

- 두 파일은 별도 구현이다.
- config.json도 별도다.
- preview.jpg도 별도다.
- README.md도 별도다.
