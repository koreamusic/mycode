# Pop WiFi MV Studio — 개발 인수인계 문서

작성일: 2026-04-25

---

## 10. Codex 인수인계 메모

현재 기준:

- 개발 목표는 PC에서 실제 제작 가능한 Pop WiFi MV Studio 구축이다.
- 모바일은 확인용이다.
- 숏츠 프리셋은 디자인 기준 원형이다.
- 롱폼 프리셋은 숏츠 프리셋을 기반으로 16:9에 맞게 재설계해야 한다.
- 단순 복사/확대/좌표 변환은 실패로 본다.
- 모든 기능은 모듈형 페이지 구조로 분리한다.
- 프로젝트 / 롱폼 인트로 / 숏츠 프리셋 / 가사 / 랜딩 / FAQ / 설정 / 렌더 큐를 각각 독립 모듈로 둔다.
- 프리셋은 사용자가 쉽게 추가/삭제/비활성화할 수 있어야 한다.
- 내부 개발 문서에는 작업 진행률, 남은 작업, 수정 파일, 다음 작업 지시를 계속 기록해야 한다.

## 11. 다음 작업 순서

1. 업로드된 숏츠 프리셋 ZIP을 기준으로 9x16 프리셋 구조를 먼저 정리한다.
2. 각 숏츠 프리셋의 색감, 액자, 제목, CTA, 하단바, 애니메이션을 분석한다.
3. 같은 디자인명을 유지한 16x9 롱폼 프리셋 초안을 별도 생성한다.
4. 프리셋 관리 페이지에서 9x16 / 16x9를 분리 노출한다.
5. FAQ 페이지에 프리셋 제작/삭제/비활성화/롱폼 재설계 규칙을 문서화한다.

## 12. 절대 금지

- 모바일 중심으로 개발하지 말 것.
- 원페이지 구조로 만들지 말 것.
- 숏츠 프리셋을 롱폼에 단순 확대 적용하지 말 것.
- 사용자가 제공한 디자인을 임의 해석해 구조를 바꾸지 말 것.
- 사이드바나 기존 주요 기능을 임의 삭제하지 말 것.
- 프리셋 추가를 위해 전체 앱 구조를 다시 깨지 말 것.
