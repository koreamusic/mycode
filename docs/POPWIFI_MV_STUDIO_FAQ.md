# Pop WiFi MV Studio — FAQ / 사용 설명

작성일: 2026-04-25
용도: 사용자와 다음 개발자가 전체 구조를 이해하기 위한 설명 문서

---

## 1. Pop WiFi MV Studio는 무엇인가?

Pop WiFi MV Studio는 PC에서 음악 기반 영상을 제작하기 위한 로컬 제작 스튜디오다.

핵심 목표는 다음과 같다.

- 싱글 MV 제작
- 롱폼 플레이리스트 영상 제작
- 숏츠 영상 제작
- 인트로 프리셋 적용
- 가사 마킹 및 검수
- 랜딩 테스트 렌더
- 최종 렌더 큐 관리

이 프로젝트는 모바일 앱이 아니다. 모바일은 결과 확인과 숏츠 화면 검수용 보조 화면이다.

---

## 2. PC 중심이라는 뜻은 무엇인가?

메인 제작 흐름은 PC에서 진행한다.

PC 기준 기능:

- 프로젝트 생성
- 음원/이미지/가사 파일 선택
- 프리셋 선택 및 미리보기
- 가사 마킹
- 랜딩 검수
- 렌더 큐 관리
- 출력 파일 관리

모바일 기준 기능:

- 숏츠 프리셋 화면 확인
- 렌더 결과 간단 확인
- 화면 깨짐 여부 검수
- 상태 확인

모바일에서 모든 제작 기능을 완성하려고 하면 안 된다.

---

## 3. 왜 원페이지 개발을 금지하는가?

기능이 많기 때문에 한 화면에 모두 넣으면 업데이트 때 구조가 쉽게 무너진다.

따라서 하나의 앱 안에서 페이지/모듈을 나누어야 한다.

필수 페이지:

1. 프로젝트
2. 롱폼 인트로 프리셋
3. 숏츠 프리셋
4. 가사 마킹
5. 랜딩 검수
6. FAQ
7. 설정
8. 렌더 큐 / 출력 관리

각 페이지는 독립적으로 수정 가능해야 하며, 공통 상태만 shared 구조로 연결한다.

---

## 4. 숏츠 프리셋과 롱폼 프리셋의 차이는?

숏츠 프리셋은 9:16 세로 화면 전용이다.
롱폼 프리셋은 16:9 가로 화면 전용이다.

둘은 같은 디자인 이름을 공유할 수 있지만, 같은 파일을 쓰면 안 된다.

예:

```text
shared/presets/9x16/moon-dust-window/
shared/presets/16x9/moon-dust-window/
```

두 프리셋은 같은 감성을 공유하지만 좌표, 크기, 여백, 화면 밀도는 별도로 설계한다.

---

## 5. 숏츠 프리셋을 롱폼에 그대로 쓰면 안 되는 이유

숏츠는 세로 집중형 화면이다.
롱폼은 가로 여백과 하단 정보 영역이 중요하다.

숏츠 좌표를 단순 확대하거나 16:9에 억지로 맞추면 다음 문제가 생긴다.

- 제목이 너무 중앙에 몰림
- CTA가 제목과 겹침
- 하단바가 어색하게 넓어짐
- 좌우 여백이 죽음
- 영상이 프레젠테이션처럼 보임
- PC 화면에서 밀도가 이상해짐

따라서 롱폼 프리셋은 반드시 재설계한다.

---

## 6. 인트로 프리셋의 기본 흐름

모든 인트로 프리셋은 아래 구조를 기본으로 한다.

```text
0~5초:
- 액자 프레임 노출
- 제목 노출

5~10초:
- 좋아요 / 구독 / 알림 CTA 노출
- 제목은 사라지거나 약해짐

10초 이후:
- 액자 프레임 사라짐
- CTA 사라짐
- 하단 투명바 노출
- 가사가 있으면 가사 표시
- 가사가 없으면 제목 표시
- 하단바 끝에는 작은 애니메이션 유지
```

이 흐름은 숏츠와 롱폼 모두 동일하다.
단, 위치와 크기는 화면비에 맞게 다르게 설계한다.

---

## 7. 프리셋은 어떻게 추가하는가?

프리셋은 다음 구조로 추가한다.

```text
shared/presets/16x9/preset-name/
  template.js
  config.json
  preview.jpg
  README.md

shared/presets/9x16/preset-name/
  template.js
  config.json
  preview.jpg
  README.md
```

필수 파일:

- template.js: 실제 렌더 코드
- config.json: 프리셋 이름, 비율, 색상, 태그, 폰트 정보
- preview.jpg: 미리보기 이미지
- README.md: 프리셋 설명과 수정 규칙

---

## 8. 프리셋은 어떻게 삭제하는가?

기본 삭제는 완전 삭제가 아니라 비활성화다.

config.json에서 다음 값을 변경한다.

```json
{
  "active": false
}
```

이렇게 하면 목록에서는 숨길 수 있지만, 나중에 복구할 수 있다.

완전 삭제는 관리자/설정 기능에서 별도 제공한다.

---

## 9. 프리셋이 마음에 안 들면 어떻게 하는가?

다음 중 하나를 선택한다.

1. 비활성화한다.
2. 새 프리셋을 추가한다.
3. 같은 이름의 새 버전을 만든다.
4. 기존 프리셋을 직접 수정하지 않고 복사본을 만든다.

추천 방식:

```text
moon-dust-window-v1
moon-dust-window-v2
moon-dust-window-v3
```

기존 안정 프리셋을 바로 덮어쓰지 않는다.

---

## 10. 랜딩 검수는 무엇인가?

랜딩 검수는 최종 렌더 전에 짧은 테스트 영상을 만들어 확인하는 단계다.

지원해야 할 검수:

- 720p 30초
- 720p 1분
- 최종 1080p 일부 구간

확인 항목:

- 인트로 타임라인
- 제목 위치
- CTA 위치
- 하단바 노출
- 가사 싱크
- 폰트 가독성
- 모바일 숏츠 화면 깨짐 여부
