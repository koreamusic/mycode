# Pop WiFi MV Studio — 상단바·사이드바 연결형 인터랙션 디자인 가이드

## 목적
이 문서는 Pop WiFi MV Studio의 상단바와 사이드바를 단순한 분리 레이아웃이 아니라, **하나의 고정된 스튜디오 프레임**처럼 느껴지게 만들기 위한 내부 디자인 기준서다.

핵심 목표는 다음과 같다.
- 심심한 관리자 화면 느낌 제거
- 상단바와 사이드바가 물리적으로 연결된 듯한 구조 구현
- React의 상태 전환과 모션을 활용해 정적 HTML 느낌 제거
- 스크롤을 늘리는 세로 누적 대신, 밀도 있고 세련된 작업 공간 유지

---

## 1. 절대 금지
- 상단바와 사이드바가 서로 무관한 직사각형처럼 따로 노는 구조
- 얇은 선 하나만 두고 연결된 것처럼 우기는 구조
- 사이드바 메뉴명을 `...`으로 자르는 구조
- 2열로 가능한 정보를 1열로 길게 세우는 구조
- 단순 hover 색상 변화만 있는 밋밋한 인터랙션
- 일반 웹사이트형 스크롤 레이아웃

---

## 2. 목표 감성
상단바와 사이드바는 다음 인상을 줘야 한다.
- **고정된 하드웨어 프레임**
- **스튜디오 콘솔**
- **맞물리는 패널 구조**
- **정교한 인터랙션을 가진 프리미엄 소프트웨어**

즉, 사용자는 “웹페이지를 본다”가 아니라 “툴을 잡고 있다”는 느낌을 받아야 한다.

---

## 3. 상단바와 사이드바의 연결 방식

### 3-1. 구조 원칙
상단바와 사이드바는 분리 요소가 아니라 하나의 Shell로 본다.

구조 예시:
- 바깥쪽: App Frame
- 안쪽 좌측: Side Rail
- 안쪽 상단: Top Command Bar
- 중앙: Workspace

### 3-2. 연결 표현 방식
다음 요소를 조합해 “맞물림”을 만든다.
- 상단바 왼쪽 시작점과 사이드바 상단 모서리가 자연스럽게 이어지는 공통 곡률
- 상단바 하단과 사이드바 우측 경계에 동일한 seam line 적용
- 상단바와 사이드바의 배경 명도 차이를 작게 두고, 경계는 단순 선이 아니라 **도킹된 접점**처럼 보이게 처리
- 접점 영역에 미세한 glow / emboss / inset shadow 적용

### 3-3. 시각 장치
허용:
- 얇은 하이라이트 라인
- 안쪽 그림자
- 접합부 notch 또는 inset shape
- active 상태에서 상단과 좌측이 함께 살아나는 연결 강조

금지:
- 과한 기어 장식
- 노골적인 산업 기계 테마
- SF HUD처럼 과도한 장식

---

## 4. 사이드바 디자인 기준

### 4-1. 사이드바는 매우 중요하다
사이드바는 단순 메뉴가 아니라 **스튜디오 상태판**이다.

반드시 포함:
- 아이콘
- 메뉴명
- 보조 설명
- 진행 상태 배지
- 현재 프로젝트 요약

### 4-2. 메뉴명 규칙
- ellipsis 금지
- 최대 2줄 자연 줄바꿈 허용
- 핵심 메뉴는 끝까지 보여야 함
- 줄 수를 줄이기 위해 폭을 확보하는 쪽을 우선

### 4-3. 아이템 인터랙션
기본 상태:
- 어두운 패널 위에 절제된 텍스트

hover 상태:
- 배경이 살짝 밀도 있게 올라옴
- 아이콘이 1~2px 정도 반응
- 텍스트 대비가 살짝 올라감

active 상태:
- 왼쪽 강조선 + 카드형 배경 + 상단바 accent와 연결된 포인트 컬러
- 상단바 상태와 시각적으로 연동

### 4-4. 프로젝트 요약 카드
하단 카드에는 최소 다음을 넣는다.
- 프로젝트명
- 타입 (싱글/플레이리스트)
- 언어
- 곡 수
- 폴더명
- 현재 단계 요약

이 카드도 2열로 갈 수 있으면 2열 유지한다.

---

## 5. 상단바 디자인 기준

### 5-1. 역할
상단바는 단순 제목줄이 아니라 **현재 작업 컨텍스트 바**다.

포함 요소:
- 현재 프로젝트명
- 현재 작업 단계
- 저장 상태
- 렌더 준비 상태
- 빠른 액션 버튼

### 5-2. 디자인 원칙
- 높이는 낮지만 밀도감 있게
- 좌측 사이드바와 시각적 접합 유지
- 버튼은 가볍게 떠 있는 느낌보다, 바 안에 정돈된 도구 버튼처럼 배치

### 5-3. 인터랙션
- 상태 pill은 살아 있어야 하지만 튀면 안 됨
- hover 시 살짝 밝아지고, 눌렀을 때는 짧은 depth 반응
- 현재 페이지와 관련된 버튼만 강조

---

## 6. React 모션 활용 기준

### 6-1. 필수
상단바/사이드바는 React의 장점을 활용해 아래를 구현한다.
- active 메뉴 전환 시 부드러운 highlight 이동
- 페이지 전환 시 중앙 작업 패널만 자연스럽게 교체
- 상단 상태 pill 변경 시 abrupt change 금지
- hover / active / selected 모두 상태 기반 애니메이션 적용

### 6-2. 추천 구현 포인트
- layout transition
- spring 기반 짧은 이동
- opacity + translate 조합
- 색상 전환 0.25~0.45초
- 과한 bounce 금지

### 6-3. 금지
- CSS만으로 뻣뻣하게 토글되는 느낌
- 요소 전체가 한 번에 깜빡이며 바뀌는 방식
- 모든 요소가 동시에 튀는 과한 모션

---

## 7. 작업 영역과의 관계

### 7-1. 고정 원칙
- 상단바 고정
- 사이드바 고정
- 중앙 작업 영역만 스크롤 또는 전환

### 7-2. 시각적 관계
- 프리뷰는 항상 중심
- 상단바/사이드바가 프리뷰를 감싸는 구조처럼 보여야 함
- 페이지를 바꿔도 Shell은 유지되고 내부 패널만 교체

---

## 8. 레이아웃 밀도 규칙
- 2열 가능하면 2열 유지
- 3열 가능한 옵션은 3열 허용
- 1열은 좁은 폭에서만 허용
- 세로 누적로 스크롤을 키우는 방식 금지
- 프로젝트 기본 설정, 폰트 선택, 랜딩 옵션은 특히 2열 우선

---

## 9. 구현 체크리스트

### 상단바
- [ ] 고정형인가
- [ ] 사이드바와 연결된 듯한 접합부가 있는가
- [ ] 현재 컨텍스트가 보이는가
- [ ] 버튼 반응이 지나치게 밋밋하지 않은가

### 사이드바
- [ ] 메뉴명이 잘리지 않는가
- [ ] ellipsis 없이 읽히는가
- [ ] active 상태가 명확한가
- [ ] 보조 설명과 상태 배지가 있는가
- [ ] 프로젝트 요약 카드가 충분한 정보를 주는가

### 인터랙션
- [ ] 메뉴 전환 시 부드러운가
- [ ] 페이지 교체가 갑작스럽지 않은가
- [ ] hover/selected 반응이 있는가
- [ ] 전체적으로 심심하지 않지만 과장되지도 않는가

---

## 10. 최종 선언
상단바와 사이드바는 장식이 아니라, Pop WiFi MV Studio의 인상을 결정하는 핵심 프레임이다.

이 둘은 반드시:
- 연결되어 보여야 하고
- 고정되어야 하며
- 정보가 충분해야 하고
- React 모션의 장점을 살려 살아 있어야 한다.

심심한 관리자 패널 같은 인상에서 벗어나,
**고급스러운 PC 전용 스튜디오 쉘**처럼 느껴지게 만드는 것이 목표다.
